# 🚩 홍대권 OTA 시장 조사 가이드라인 (Track B Pilot)

본 문서는 홍대권 상권의 OTA(Agoda, Booking, Airbnb) 리스팅 수집 시 준수해야 할 정합성 기준을 정의합니다.

## 1. 수집 표준 파라미터
모든 시장 데이터는 아래 파라미터가 고정된 상태에서 수집되어야 합니다.

| 파라미터 | 기준값 | 비고 |
| :--- | :--- | :--- |
| **인원 (Guests)** | 2명 | 성인 기준 |
| **박수 (Nights)** | 1박 | 요일별 편차 고려 |
| **체크인 날짜** | 수집일 + 14일 | 예약 안정성이 높은 시점 선택 |
| **통화 (Currency)** | KRW (원화) | 환율 변동 최소화 |

## 2. 수집 대상 정보 (Field Definition)
각 리스팅에서 반드시 확보해야 할 정보는 다음과 같습니다.

- **총 가격 (price_total_krw)**: 모든 세금과 봉사료가 포함된 **최종 결제 금액**. (세금 불포함 가격 수집 시 주의)
- **객실명 (room_type_name)**: OTA에 표기된 실제 객실 명칭 (예: 'Deluxe Double', '6-Bed Mixed Dorm').
- **수용 인원 (max_guests)**: 해당 객실의 최대 투숙 가능 인원.
- **침대 구성 (bed_type_raw)**: 침대 개수 및 종류 (예: '1 Queen Bed', '2 Bunk Beds').
- **욕실 여부 (private_bathroom_yn)**: 객실 내 전용 욕실 여부 (Y/N).
- **무료 취소 (refundable_yn)**: 수집 시점 기준 무료 취소 가능 여부 (Y/N/NA).
- **조식 포함 (breakfast_yn)**: 조식 포함 여부 (Y/N/NA).
- **순위 (source_rank)**: 검색 결과 리스트에서 몇 번째로 노출되었는지 기록.

## 3. 지역 검색 방법
- 특정 숙소명을 검색하는 대신 **"홍대"**, **"Hongdae"**, 또는 **홍대입구역 좌표 기반 반경 검색**을 수행합니다.
- 상위 노출되는 리스팅부터 순차적으로 수집하여 시장의 '인기/기준' 숙소 데이터를 확보합니다.

## 4. 데이터 표준화 (Standardization)
분석의 일관성을 위해 다음 필드는 표준화된 값을 할당합니다.

### 4.1 객실 유형 (`room_type_std`)
- `dorm_bed`: 도미토리 내 침대 1개 단위 판매
- `dorm_room`: 도미토리 객실 전체 (그룹용)
- `private_room`: 일반적인 개인실 (게스트하우스 등)
- `hotel_room`: 호텔형 객실
- `suite`: 스위트룸
- `family_room`: 다인 가족실
- `unknown`: 분류 불가

### 4.2 욕실 타입 (`bathroom_type`)
- `private`: 객실 내 전용 욕실
- `shared`: 공용 욕실 사용
- `unknown`: 정보 없음

## 5. 데이터 검수 규칙
- 이름이 영문으로만 되어 있는 경우, 가능한 국문명과 함께 기록합니다.
- `raw_listing_url`은 상세 페이지로 들어갈 수 있는 고유 링크여야 합니다.
- 매진(`sold_out`)인 경우 가격은 0으로 기록하되 상태를 명시합니다.
- **좌표 정보**: `raw_lat`, `raw_lng`가 있는 경우 반드시 누락 없이 수집합니다. (서울시 마스터 매칭의 핵심)
